# Telnet

## Overview

Telnet was one of the earliest remote access protocols developed to allow users to connect to and manage systems over a network. It follows a client-server architecture and enables users to execute commands on a remote machine as if they were physically present at that system.

Although Telnet played a major role in the development of remote administration, it is now largely replaced by SSH due to security concerns.

---

# What is Telnet?

**Telnet (Telecommunication Network)** is a network protocol that allows users to remotely access and manage another computer over a network using a command-line interface.

It establishes a connection between:

- Telnet Client
- Telnet Server

Once connected, users can execute commands on the remote system.

## Definition

> Telnet is a remote login protocol that allows a user to access and control a remote computer over a network through a command-line interface.

---

# Purpose of Telnet

Telnet is primarily used for:

- Remote system administration
- Accessing network devices
- Managing servers remotely
- Troubleshooting network services
- Testing connectivity to specific ports

---

# Telnet Client-Server Architecture

## Components

### 1. Telnet Client

The client is installed on the user's machine and is responsible for:

- Sending commands to the server
- Receiving responses
- Displaying output to the user

### 2. Telnet Server

The server runs on the remote machine and is responsible for:

- Accepting Telnet connections
- Processing user requests
- Executing commands
- Returning results

---

# How Telnet Works

## Step 1: User Enters a Command

The user types a command in the terminal.

```bash
telnet 192.168.1.10
```

---

## Step 2: Telnet Client Receives Input

The Telnet client captures the user's keystrokes and commands.

---

## Step 3: Conversion into Standard Format

The command is converted into a standard character format that both client and server can understand.

This allows communication even if:

- Client and server use different operating systems
- Client and server use different hardware architectures

---

## Step 4: Data Sent Through TCP/IP

The command is encapsulated and sent through:

- Application Layer
- Transport Layer (TCP)
- Network Layer (IP)
- Data Link Layer
- Physical Layer

---

## Step 5: Telnet Server Receives Request

The remote Telnet server receives the request through its TCP/IP stack.

---

## Step 6: Command Interpretation

The Telnet server passes the request to the operating system.

The operating system interprets the command and executes it.

---

## Step 7: Application Execution

If the command requires an application or service, the operating system invokes the appropriate program.

Examples:

- Opening files
- Listing directories
- Checking processes
- Managing services

---

## Step 8: Response Returned

The output generated by the operating system is returned through the Telnet server to the client.

The user then sees the result on their screen.

---

# Universal Character Representation

One challenge in remote communication is that client and server systems may use different operating systems.

For example:

- Windows
- Linux
- Unix

To ensure both sides understand each other, Telnet converts communication into a standard format before transmission.

This makes communication possible regardless of the operating systems involved.

---

# Limitations of Telnet

The biggest problem with Telnet is security.

## No Encryption

Telnet sends everything in plain text, including:

- Usernames
- Passwords
- Commands
- Responses

Anyone intercepting the traffic can read the data.

---

## Security Risks

Because data is transmitted in clear text:

- Credentials can be stolen.
- Sessions can be monitored.
- Attackers can capture sensitive information.

This is why Telnet is considered insecure for modern environments.

---

# Why Telnet is Considered Outdated

Modern networks require:

- Confidentiality
- Integrity
- Authentication

Telnet does not provide these security features.

Therefore, it is rarely used on the public internet today.

---

# SSH - Secure Alternative to Telnet

## What is SSH?

**SSH (Secure Shell)** is a secure network protocol used for remote login and remote system administration.

It performs the same core functions as Telnet but with strong security controls.

---

# Advantages of SSH

## Encryption

SSH encrypts all communication between client and server.

This protects:

- Usernames
- Passwords
- Commands
- Files
- Session data

---

## Authentication

SSH supports:

- Password Authentication
- Public Key Authentication
- Multi-Factor Authentication (MFA)

---

## Data Integrity

SSH ensures that transmitted data is not altered during transit.

---

## Secure Remote Administration

System administrators can safely manage servers over the internet.

---

# Telnet vs SSH

| Feature | Telnet | SSH |
|----------|---------|---------|
| Security | Insecure | Secure |
| Encryption | No | Yes |
| Authentication | Basic | Strong |
| Password Protection | No | Yes |
| Internet Usage | Rare | Common |
| Port Number | TCP 23 | TCP 22 |
| Data Transmission | Plain Text | Encrypted |

# Local Login

## Definition

A Local Login occurs when a user accesses and uses a computer directly without connecting to another machine over a network.

The user interacts with:

- Local Operating System
- Local Files
- Local Applications
- Local Resources

---

## Examples

- Opening videos
- Reading PDF files
- Accessing folders
- Running applications
- Viewing local documents

Even when the internet is disconnected, these activities can still be performed because they rely on the local system.

---

```text
User
  │
  ▼

  │
  ▼

  │
  ▼

```

---

# Local Login Process

1. User enters input.
2. Operating System receives the request.
3. OS interprets the request.
4. Required application is launched.
5. Result is displayed to the user.

---

# Remote Login

## Definition

A Remote Login occurs when a user accesses another computer over a network or the internet.

Instead of working on the local machine, commands are executed on the remote system.

---

# Remote Login Requirements

- Network connectivity
- Client software
- Remote server
- Authentication credentials

---
