# Security Posture

Security Posture refers to an organization's overall ability to protect its systems, networks, applications, users, and data from security threats.

A strong security posture includes:

* Security controls
* Risk management processes
* Incident response plans
* Security awareness training
* Continuous monitoring

The goal is to reduce security risks while protecting critical assets and business operations.

# CISSP Security Domains

## 1. Security and Risk Management

This domain focuses on:

* Security goals and objectives
* Risk management
* Risk mitigation
* Compliance
* Legal and regulatory requirements
* Business continuity

### Risk Mitigation

Risk mitigation is the process of reducing the likelihood or impact of a security risk.

Examples:

* Maintaining backups
* Using antivirus solutions
* Applying security patches
* Employee security awareness training

### Compliance

Compliance ensures that organizations follow legal, regulatory, and industry standards.

Examples:

* GDPR
* HIPAA
* PCI-DSS
* ISO 27001

### Business Continuity

Business Continuity ensures that critical business operations continue during and after disruptions.

Examples:

* Disaster Recovery Plans (DRP)
* Backup infrastructure
* Alternate data centers
* Incident response planning

## 2. Asset Security

Asset Security focuses on protecting organizational data throughout its lifecycle.

Activities include:

* Data classification
* Data storage
* Data handling
* Data retention
* Secure destruction

### Sensitive Data Examples

* PII (Personally Identifiable Information)
* SPII (Sensitive Personally Identifiable Information)

Examples:

* Aadhaar Number
* PAN Number
* Passport Information
* Banking Details

---

## 3. Security Architecture and Engineering

This domain focuses on designing and implementing secure systems, technologies, and security controls.

Objectives:

* Secure infrastructure design
* Secure system implementation
* Protection of organizational assets
* Defense against cyber threats

### Shared Responsibility

Security is not only the responsibility of the security team.

Every employee plays a role in maintaining organizational security.

---

## 4. Communication and Network Security

This domain focuses on securing networks and protecting data during transmission.

Examples:

* Firewalls
* VPNs
* IDS (Intrusion Detection Systems)
* IPS (Intrusion Prevention Systems)
* Network Segmentation

Goal:

Protect data while it is being transmitted across networks.

---

## 5. Identity and Access Management (IAM)

IAM ensures that the right individuals have access to the right resources at the right time.

### Identification

The user claims an identity.

Example:

```text
Username: roshni
```

### Authentication

The user proves their identity.

Examples:

* Password
* OTP
* Fingerprint
* Face Recognition

### Authorization

Determines what actions a user is allowed to perform.

Example:

* Employee → Read Access
* Administrator → Read, Write, Delete Access

### Accountability

Tracks and records user actions.

Examples:

* Login Logs
* Audit Logs
* Event Logs

---

## 6. Security Assessment and Testing

This domain focuses on evaluating security controls and identifying weaknesses.

Activities include:

* Security audits
* Vulnerability assessments
* Penetration testing
* Security control testing

Goal:

Identify weaknesses before attackers do.

---

## 7. Security Operations

Security Operations focuses on monitoring, detecting, investigating, and responding to security incidents.

Activities include:

* Threat monitoring
* Incident response
* Security investigations
* Preventive security measures

SOC Analysts primarily work within this domain.

---

## 8. Software Development Security

This domain focuses on building secure software throughout the Software Development Lifecycle (SDLC).

### Design Phase

* Secure design reviews
* Threat modeling

### Development Phase

* Secure coding practices
* Code reviews

### Testing Phase

* Security testing
* Vulnerability assessments

### Deployment Phase

* Penetration testing
* Security validation
